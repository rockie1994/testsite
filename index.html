404 to Root: How a Forgotten Subdomain Led to Server Takeover

During a routine security assessment, I discovered an overlooked subdomain that eventually granted me complete control over the organization's main server. Here's how it happened.

The Initial Discovery

While enumerating subdomains for example.com, I noticed legacy.example.com returning a 404 error. Most would move on, but 404 pages can sometimes reveal more than intended.

The seemingly innocent 404 page that started it all

Following the Trail

The 404 page included a reference to /static/js/main.js that shouldn't have been publicly accessible. Fetching this file revealed:

// Configuration snippet from main.js
const API_CONFIG = {
    baseUrl: 'http://internal-api.legacy.example.com',
    authToken: 'Bearer xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx'
};

Key Finding: The JavaScript file exposed internal API endpoints and what appeared to be a valid authentication token.

Exploiting the Findings

Using the discovered token with the internal API endpoint:

curl -H "Authorization: Bearer xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx" \
http://internal-api.legacy.example.com/v1/admin/users

To my surprise, this returned a full list of system users with their roles and permissions.

The Pivotal Mistake

The organization had:

  1. Failed to properly decommission the legacy subdomain
  2. Left development configuration in production
  3. Used the same authentication tokens across environments

Path to Root Access

With admin access to the internal API, I was able to:

POST /v1/admin/scripts/execute
{
    "command": "whoami",
    "timeout": 5000
}

The API executed system commands directly on the server, leading to full compromise.

Lessons Learned

  • Properly decommission old subdomains and services
  • Never include sensitive configuration in client-side code
  • Implement proper authentication segregation between environments
  • Regularly audit exposed endpoints and error pages

This vulnerability was responsibly disclosed and patched within 24 hours. The organization has since implemented a comprehensive audit of all their digital assets.
